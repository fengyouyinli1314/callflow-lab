# 系统总体设计

## 一、系统定位

callflow-lab 是“复杂外呼任务对话模型自动评测平台”。本系统不是训练对话模型的平台，也不是实际外呼业务系统；它面向已有对话模型，评测模型在复杂外呼任务指令、多轮上下文、用户打断和业务约束下的表现。

系统提供从任务指令导入、测试用例生成、用户模拟器 Agent、被测模型接入、规则评分、LLM 评估、可解释报告到批量统计的完整评测链路。当前默认脱敏任务包括“飞毛腿骑手合同生效外呼评测”和“课程直播产品升级外呼评测”，后续可以继续导入新的复杂外呼任务。

## 二、设计目标

- 支持导入复杂任务指令，并解析 Role、Task、Opening Line、Call Flow、Knowledge Points、Constraints 等结构化信息。
- 支持基于任务生成或管理多轮测试用例，覆盖不同被外呼对象画像。
- 支持用户模拟器 Agent 动态生成用户发言，模拟拒绝、追问、打断、情绪不满、开车等真实对话行为。
- 支持接入 `mock_fallback` 兜底、OpenAI compatible API 或外部 custom endpoint 作为被测对话模型。
- 支持规则评分和 LLM-as-a-Judge 混合评估，输出量化分数、规则分、Judge 分、融合分、失败规则、证据轮次和优化建议。
- 支持规则追溯，按 global / case / stage / triggered / pending / untriggered 等来源解释规则为什么激活、待完成或不参与扣分。
- 支持单条问答检测，在不生成正式报告的情况下快速调试某个用户问题的模型回答和规则命中。
- 支持批量评测，按任务、用例、模型 provider 和重复次数生成多份 run/report，并汇总平均分、通过率和失败规则 TOP5。

## 三、总体架构

系统采用前后端分离架构：

- 前端层：Vue 3 + Element Plus + ECharts，负责任务管理、用例查看、单次评测、批量评测、模型配置、报告展示和数据大屏。
- 接口层：FastAPI 提供 REST API，并通过 Swagger 展示接口文档。
- 业务编排层：`EvaluationService` 负责单次评测主流程；`BatchEvaluationService` 负责批量展开和汇总。
- Agent 层：`UserSimulatorAgent` 负责被外呼对象模拟；`EvaluatorAgent` 负责语义级评估。
- 模型接入层：`TargetModelClient` 统一接入被测模型，并在返回前执行任务类型校验和去重。
- 评分层：`RuleJudge` 执行硬规则检查，`EvaluatorAgent` 生成六项指标评估。
- 数据层：SQLModel + SQLite 存储任务、用例、run、message、report、batch run 和 batch item。

## 四、核心流程

平台核心流程为：

复杂任务指令导入
→ 解析 Role / Task / Opening Line / Flow / Knowledge / Constraints
→ 自动生成测试用例
→ 用户模拟器 Agent 发起多轮对话
→ 被测模型回复
→ 规则评分 + LLM 评估
→ 生成可解释报告
→ 批量统计评测结果

在单次评测中，用户选择任务和用例后，后端创建 run；每一轮先由用户模拟器 Agent 根据任务、画像和历史对话生成用户发言，再由 `TargetModelClient.generate_reply(...)` 调用被测模型生成回复。系统保存响应时间、命中规则、失败规则和对话状态。对话结束后生成报告。

在批量评测中，系统按任务、用例范围、模型 provider 和重复次数展开多个单次评测，复用同一条单次评测链路，并汇总批量结果。

在单条问答检测中，系统直接接收一条或多条用户发言，调用 `TargetModelClient.generate_reply(...)` 生成回答，并使用轻量事实检查和规则评分返回单条结果。该流程不创建正式 run/report，只用于话术调试和演示排查。

## 五、主要模块

### 任务与用例模块

任务模块保存复杂外呼指令、任务类型、数据来源和结构化片段。用例模块保存用户画像、初始问题、最大轮数、期望目标、必须满足规则和禁止触发规则。当前默认评测数据来自脱敏 Excel 指令。

### 用户模拟器 Agent

用户模拟器 Agent 根据 `task.instruction_text`、`task.task_type`、`case.user_profile`、`case.initial_message`、规则约束、历史消息和当前轮次生成下一轮用户发言。它维护情绪、耐心、目标进度和当前意图，用于模拟不同被外呼对象。

### 被测模型接入

被测模型回复统一由 `TargetModelClient.generate_reply(...)` 生成。系统支持：

- `mock_fallback`：本地兜底模型，仅用于无 API Key 或演示环境不可用时保证流程跑通，不作为系统 AI 能力亮点。
- `openai_compatible`：真实大模型 API 接入，可接入 Qwen、DeepSeek、GPT、智谱、Moonshot 等兼容 OpenAI 格式的模型。
- `custom_endpoint`：自定义被测模型接口，可接入外部被测模型服务、企业内部模型或其他团队接口。



### 评分与报告模块

规则评分负责硬约束检查，包括必达流程、禁止规则、字数限制、业务串场、忽略用户打断、安全结束、回答用户问题和机械重复。评估器 Agent 读取任务指令、测试目标、对话记录和规则评分结果，输出六项指标：

- 任务完成度
- 指令遵循率
- 外呼流程覆盖率
- 约束遵守率
- 上下文一致性
- 回复质量

报告页展示总分、指标分、评分公式、雷达图、命中规则、失败规则、规则追溯、评审结论、扣分原因、证据轮次、对话片段、优化建议和完整对话记录。

每项指标保留三类分数来源：

```text
combined_score = rule_score * 0.7 + judge_score * 0.3
```

规则追溯记录每条规则的来源、状态、激活轮次、激活原因、证据文本和扣分原因。后续流程节点展示为 pending，不会在当前轮直接计入失败；未触发规则展示为 untriggered，不参与扣分。

### 批量评测模块

批量评测支持选择多个任务、用例范围、一个或多个模型 provider 和重复次数。系统生成多个 run/report，并返回：

- 平均分
- 平均响应时间
- 通过率
- 任务得分汇总
- 指标得分汇总
- 失败规则 TOP5
- 报告列表
- 可选的模型 provider 对比统计

批量评测不是强制 A/B 测试；多 provider 只作为扩展统计能力。

## 六、 Agent 使用边界

当前系统不进行模型训练。原因是：




当前 AI 能力来自：
- 系统采用 LLM Agent 编排、轻量 RAG 和 LLM-as-a-Judge 实现 AI 能力，重点验证被测模型在多轮任务中的表现。
- 用户模拟器 Agent 动态生成多轮用户行为，根据任务指令、用户画像、历史对话和用户状态推进追问、打断、接受或终止。
- Knowledge Retriever 根据用户问题召回官方知识点，把 Opening Line、Call Flow、FAQ 和 Constraints 作为本轮生成与评估依据。
- `openai_compatible` 接入真实大模型，`custom_endpoint` 接入外部被测模型服务或企业内部模型。
- LLM-as-a-Judge 自动评分并给出证据、扣分原因和改进建议。
- Report Agent 自动生成可解释、可量化的评测报告。

`mock_fallback` 只是本地演示兜底，用于无 API Key 或演示环境不可用时保证流程跑通，不作为系统 AI 能力亮点。真实模型接入方式是 `openai_compatible` 和 `custom_endpoint`。

## 七、数据模型概览

- `evaluation_tasks`：评测任务，保存任务指令、结构化片段、任务类型和数据来源。
- `evaluation_cases`：测试用例，保存用户画像、初始问题、期望目标和规则约束。
- `evaluation_runs`：单次评测执行记录，保存任务、用例、模型 provider、状态和总分。
- `run_messages`：每轮对话记录，保存用户发言、模型回复、延迟、轮次评分和 metadata。
- `evaluation_reports`：评测报告，保存指标分、失败规则、LLM 评估、证据链和完整对话。
- `evaluation_batch_runs`：批量评测记录，保存批量配置和汇总结果。
- `evaluation_batch_run_items`：批量子任务记录，关联具体 run/report。

## 八、接口概览

- `/api/tasks`：任务管理。
- `/api/cases`：测试用例管理。
- `/api/runs/start`：启动单次评测。
- `/api/runs/stream`：启动流式单次评测，逐步返回用户发言、被测模型回复、规则结果和报告生成状态。
- `/api/runs/quick-check`：执行单条问答检测，不创建正式 run/report。
- `/api/runs/{run_id}/messages`：查看单次评测对话。
- `/api/reports/{report_id}`：查看完整报告。
- `/api/batch-runs/start`：启动批量评测。
- `/api/batch-runs/{batch_id}`：查看批量详情。
- `/api/batch-runs/{batch_id}/summary`：查看批量汇总。
- `/api/model-providers`：查看模型 provider 配置。
- `/api/model-providers/test`：测试 provider 配置可用性。
- `/api/dashboard/summary`：数据大屏统计。

## 九、商业价值

- 降低人工评估成本：自动生成对话、评分和报告，减少人工逐轮阅读和打分。
- 提高外呼模型回归测试效率：支持同一任务在多个用例、多个版本或多个 provider 下重复评测。
- 支持复杂任务指令下的自动化验证：能检查流程覆盖、约束遵守、上下文一致性和回复质量。
- 发现高风险问题：能定位漏流程、违反约束、回复冗长、答非所问、机械重复和跨业务串场等问题。
- 支持后续多模型、多版本批量评测：为模型选型、版本发布前回归和质量监控提供量化依据。
