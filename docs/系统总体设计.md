# 系统总体设计

## 项目背景

复杂指令下的多轮对话评测，需要同时考察模型是否理解业务场景、是否遵循流程约束、是否能在多轮上下文中保持一致，并在涉及权益、退款、变更等敏感动作时避免越权承诺。callflow-lab 面向客服、人工外呼、业务助手等场景，提供可配置、可执行、可解释的评测平台。

## 需求理解

系统不仅要给出一个最终分数，还要说明模型为什么得分、哪里扣分、对应哪一轮对话、应如何优化。因此平台需要把“任务指令、测试用例、用户模拟、被测回复、规则判断、模型评分、报告解释”串成完整链路。

## 系统目标

- 支持创建复杂评测任务和多轮测试用例。
- 支持用户模拟器自动推动对话。
- 支持被测客服模型 mock 和真实模型接口预留。
- 支持规则评分与模型评分两类评分机制。
- 支持输出量化、可解释、可复盘的评测报告。
- 支持比赛演示中的快速启动、稳定运行和可视化展示。

## 总体架构

系统采用前后端分离架构：

- 前端层：Vue 3 + Element Plus + ECharts，负责任务管理、用例筛选、评测执行和报告展示。
- 接口层：FastAPI 提供 REST API，并通过 Swagger 自动生成接口文档。
- 业务层：评测服务编排用户模拟器、被测模型、规则评分、模型评分和报告生成。
- 数据层：SQLModel + SQLite 存储任务、用例、执行记录、对话消息和评测报告。
- 模型接入层：`llm_client.py` 统一封装 mock 与 OpenAI compatible API。

## 核心业务流程

1. 用户在前端选择评测任务和测试用例。
2. 后端创建评测执行记录。
3. 用户模拟器根据用户画像、初始问题、历史对话生成下一轮用户发言。
4. 被测模型模拟器根据场景和用户发言生成客服回复。
5. 系统保存每一轮用户发言、客服回复、响应时间和规则命中情况。
6. 对话结束后，规则评分模块计算五项核心指标。
7. 模型评分模块输出稳定格式的 mock 评审结果，真实模型接口可按配置切换。
8. 报告模块生成包含指标解释、失败案例、证据片段和优化建议的评测报告。
9. 前端展示完整报告和数据大屏统计。

## 用户模拟器设计

用户模拟器基于测试用例中的 `user_profile`、`initial_message`、`expected_goals`、`difficulty` 和历史对话生成下一轮发言。当前支持：

- 普通用户：表达清晰，关注处理结果和时效。
- 情绪激动用户：强调不满和紧迫感，考察模型安抚能力。
- 反复追问用户：持续确认处理条件、时效和限制。
- 信息缺失用户：缺少订单号、券码等关键信息，考察模型引导能力。
- 需求变更用户：中途改变诉求，考察上下文理解和风险说明能力。

## 自动评分模块设计

规则评分模块围绕以下指标计算：

- 任务完成度：必须规则和期望目标的覆盖程度。
- 指令遵循率：必须规则是否执行，禁止规则是否规避。
- 上下文一致性：多轮回复是否承接用户新增信息。
- 安全合规性：是否出现越权承诺、绕过平台等风险。
- 回复质量：是否清晰、可执行、有安抚和下一步动作。
- 平均响应时间：基于每轮模拟响应耗时计算。

模型评分模块默认返回 mock 结果，并通过 `llm_client.py` 预留 OpenAI compatible API 接入。

## 评测报告设计

报告包含：

- 总分和五项指标得分。
- 平均响应时间和对话轮数。
- 失败规则列表。
- 每个指标的扣分原因、证据轮次、对话片段和优化建议。
- `failure_cases`：逐条列出失败规则、严重级别、证据轮次、证据描述和建议。
- `explainability`：总体原因、分数拆解、关键发现和改进建议。

## 数据库设计

- `evaluation_tasks`：评测任务，保存场景、系统指令和评测目标。
- `evaluation_cases`：测试用例，保存用户画像、初始问题、期望目标和规则约束。
- `evaluation_runs`：评测执行记录，保存状态、总分和完成时间。
- `run_messages`：每轮对话记录，保存用户发言、客服回复、延迟和轮次评分。
- `evaluation_reports`：评测报告，保存指标、失败规则、解释性字段和完整对话。

## 接口设计概览

- `/api/tasks`：任务增删改查。
- `/api/cases`：测试用例增删改查，支持按任务筛选。
- `/api/runs/start`：启动自动评测。
- `/api/runs/{run_id}`：查看执行记录。
- `/api/runs/{run_id}/messages`：查看完整对话。
- `/api/reports`：报告列表。
- `/api/reports/{report_id}`：报告详情。
- `/api/reports/{report_id}/summary`：报告摘要。
- `/api/dashboard/summary`：数据大屏统计。
