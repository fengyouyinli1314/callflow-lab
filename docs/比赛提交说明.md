# 比赛提交说明

## 项目简介

callflow-lab 是一个面向复杂指令下多轮对话场景的评测系统。系统通过可配置任务、测试用例、用户模拟器、被测模型模拟器、规则评分、模型评分和可解释报告，评估对话模型在客服类复杂任务中的任务完成度、指令遵循率、上下文一致性、安全合规性和回复质量。

## 创新点

- 以多轮对话流程而非单轮问答作为评测对象，更贴近真实客服业务。
- 用户模拟器支持情绪激动、反复追问、信息缺失、需求变更等复杂用户行为。
- 评分报告不仅输出总分，还输出证据轮次、对话片段、失败规则和优化建议。
- 默认 mock 模式即可稳定演示，同时通过统一模型客户端预留真实模型接入。
- 数据大屏集中展示评测次数、平均得分、响应延迟、失败规则和趋势变化。

## 核心功能

- 评测任务管理：配置业务场景、复杂系统指令和评测目标。
- 测试用例管理：配置用户画像、初始问题、最大轮数和规则约束。
- 自动评测执行：自动完成多轮用户模拟和客服模型回复。
- 规则评分：基于必须规则、禁止规则和对话质量计算指标。
- 模型评分预留：支持 mock 和 OpenAI compatible API。
- 可解释报告：展示五项指标、扣分原因、证据片段和优化建议。
- 前端演示台：支持任务查看、用例筛选、开始评测、报告查看和大屏统计。

## 演示流程

1. 启动后端并访问 Swagger，确认接口可用。
2. 启动前端并进入数据大屏，查看系统整体统计。
3. 进入评测任务页面，查看三个示例任务。
4. 进入测试用例页面，按任务筛选用例。
5. 进入开始评测页面，选择任务和测试用例。
6. 点击“开始评测”，观察多轮对话生成和轮次评分。
7. 评测完成后点击“查看报告”，查看总分、雷达图、失败规则、失败案例和优化建议。

## 交付文件说明

- `backend/`：FastAPI 后端服务、数据库模型、API、评测服务、评分模块和测试用例。
- `frontend/`：Vue 3 前端页面、路由、接口配置和可视化组件。
- `docs/系统总体设计.md`：系统背景、架构、流程和模块设计。
- `docs/接口说明.md`：全部后端接口、请求参数和返回示例。
- `docs/测试说明.md`：功能测试、接口测试和评测准确性测试说明。
- `docs/部署说明.md`：本地环境、启动命令、模型配置和常见问题。
- `docs/比赛提交说明.md`：项目简介、创新点、核心功能、演示流程和后续方向。
- `README.md`：项目总览和快速启动说明。

## 后续优化方向

- 接入真实被测模型和独立裁判模型，支持多模型横向对比。
- 增加规则配置页面，使业务规则可视化维护。
- 引入更细粒度的对话状态机，支持复杂流程分支和中断恢复。
- 增加批量评测、报告导出和历史版本对比。
- 增强评分标注闭环，将人工复核结果用于优化评分规则。
