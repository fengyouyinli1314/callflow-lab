# 比赛提交说明

## 一、项目简介

callflow-lab 是“复杂外呼任务对话模型自动评测平台”。本系统不是训练对话模型，而是评测已有对话模型在复杂外呼任务指令下的表现。平台面向外呼质检、对话模型回归测试和模型选型场景，提供任务指令导入、用户模拟器 Agent、被测模型接入、规则评分、LLM 评估和可解释报告生成能力。

当前默认评测集来自脱敏 Excel 任务，包括飞毛腿骑手合同生效外呼评测和课程直播产品升级外呼评测。系统设计不写死这两个任务，后续可以继续导入新的复杂外呼任务指令。

## 二、比赛要求对应关系

比赛关注两个核心点：

1. 构建用户模拟器，充分有效测试对话模型在特定任务指令下的效果。
2. 自动产出评测报告，评测过程可解释、结果可量化。

本项目对应实现：

- 用户模拟器 Agent：根据任务指令、用户画像、初始问题和历史对话动态生成用户发言，支持追问、拒绝、情绪不满、开车、忙碌、技术不熟悉等行为。
- 自动评测链路：自动执行多轮对话，并记录每轮用户状态、模型回复、响应时间和规则命中情况。
- 混合评分体系：规则评分负责硬约束检查，评估器 Agent 负责语义级评分。
- 可解释报告：展示总分、六项指标分、评分公式、失败规则、证据轮次、对话片段、扣分原因和优化建议。
- 批量评测：支持多个任务、多个用例、多个 provider 和重复次数的批量执行，输出平均分、通过率和失败规则 TOP5。

## 三、核心流程

系统核心流程为：

复杂任务指令导入
→ 解析 Role / Task / Opening Line / Flow / Knowledge / Constraints
→ 自动生成测试用例
→ 用户模拟器 Agent 发起多轮对话
→ 被测模型回复
→ 规则评分 + LLM 评估
→ 生成可解释报告
→ 批量统计评测结果

单次评测适合演示某个用例的完整多轮对话和证据链；批量评测适合评估一组任务和用例下模型整体表现。

## 四、功能清单

- 评测任务管理：展示复杂任务指令、结构化片段、任务类型和数据来源。
- 测试用例管理：展示用户画像、初始问题、难度、最大轮数、必须满足规则和禁止触发规则。
- 开始评测：选择任务、用例和模型 provider，自动执行多轮对话。
- 用户模拟器 Agent：模拟不同被外呼对象行为，并根据被测模型回复动态推进或结束。
- 被测模型接入：支持 `mock_fallback`、`openai_compatible` 和 `custom_endpoint`；`mock_fallback` 仅用于兜底演示，真实 AI 能力主要来自外部大模型或自定义被测模型接口。
- 规则评分：检查流程覆盖、禁止规则、字数限制、业务串场、用户打断、安全结束、回答问题和机械重复。
- LLM 评估器：输出任务完成度、指令遵循率、外呼流程覆盖率、约束遵守率、上下文一致性和回复质量。
- 报告页：展示指标、证据、失败案例、建议和完整对话。
- 批量评测：按任务、用例、provider 和重复次数生成多个 run/report，并输出统计结果。
- 模型配置页：展示 provider 类型、启用状态、Base URL、Model Name、Endpoint 和连接测试入口，不展示 API Key 明文。
- 数据大屏：展示任务数、用例数、评测次数、批量次数、平均得分、平均通过率、失败规则 TOP5 和最近批量结果。

## 五、深度学习与 Agent 使用边界

本系统当前不直接训练大模型，而是通过 LLM API、Prompt 工程和轻量级 Agent 编排实现用户模拟和自动评估。

用户模拟器 Agent 负责模拟不同被外呼对象的行为；评估器 Agent 负责结合任务指令和对话记录进行语义级评分；被测模型主要来自真实外部对话模型接口，`mock_fallback` 只用于演示兜底。

后续如提供人工标注对话数据，可进一步微调用户模拟器或评估器模型，让模拟用户更接近真实外呼对象，让评分结果更接近人工专家判断。

## 六、创新点

- 从单轮问答扩展到多轮外呼任务评测，覆盖流程推进、用户打断和状态变化。
- 用户模拟器具备状态记忆，能根据模型是否回答问题、是否重复、是否违反约束调整下一轮发言。
- 被测模型回复有任务类型白名单校验，能防止不同业务任务之间回复串场。
- 混合评测体系兼顾硬规则和语义质量，报告同时可量化和可解释。
- 批量评测让系统从单条 Demo 升级为可用于回归测试和模型选型的平台。

## 七、商业价值

- 降低人工评估成本：减少人工逐轮模拟用户、阅读对话和打分的工作量。
- 提高外呼模型回归测试效率：模型版本升级后，可以批量跑固定任务集，快速发现质量变化。
- 支持复杂任务指令自动化验证：检查模型是否覆盖必达流程、遵守禁止约束、响应用户追问。
- 发现典型质量问题：包括漏流程、违反约束、回复冗长、答非所问、机械重复和业务串场。
- 支持多模型、多版本批量评测：后续可用于模型选型、版本发布门禁和持续质量监控。

## 八、演示建议

1. 打开数据大屏，查看整体评测次数、平均得分、平均通过率和失败规则 TOP5。
2. 进入评测任务详情，查看脱敏任务的 Role、Task、Opening Line、Call Flow、Knowledge Points 和 Constraints。
3. 在测试用例页按任务筛选，查看用户画像、初始问题和规则约束。
4. 在开始评测页选择任务、用例和 provider，运行一次多轮评测。
5. 打开完整报告，查看总分、雷达图、失败规则、证据链、扣分原因和优化建议。
6. 在批量评测页选择两个默认任务，使用 `mock_fallback` 跑一次兜底演示评测，或配置 `openai_compatible` / `custom_endpoint` 后接入真实被测模型，查看平均分、通过率、失败规则 TOP5 和报告列表。
7. 在模型配置页查看四类 provider，说明真实模型可通过 `.env` 配置接入，前端不展示 API Key。
