# 接口说明

基础地址：

```text
http://127.0.0.1:8080
```

## 系统健康检查

`GET /health`

返回示例：

```json
{
  "status": "ok",
  "service": "callflow-lab"
}
```

## 评测任务

`POST /api/tasks`

请求示例：

```json
{
  "name": "飞毛腿骑手合同生效外呼评测",
  "description": "评测被测模型能否围绕合同生效、配送要求、排名规则和退出流程完成外呼沟通。",
  "target_scenario": "飞毛腿骑手合同生效",
  "system_instruction": "说明单日/多日完成要求、合同与派单影响，回答排名、天气、退出和奖励等问题，禁止承诺一定获得资格。",
  "evaluation_goal": "评测复杂外呼指令遵循、用户问题回应和合规边界。"
}
```

`GET /api/tasks`：获取任务列表。

`GET /api/tasks/{task_id}`：获取任务详情。

`PUT /api/tasks/{task_id}`：更新任务。

`DELETE /api/tasks/{task_id}`：删除任务。

## 测试用例

`POST /api/cases`

请求示例：

```json
{
  "task_id": 1,
  "name": "骑手质疑报名排名",
  "user_profile": "情绪不满骑手",
  "initial_message": "为什么别人能报上，我报不上？",
  "expected_goals": ["说明报名按排名", "说明不是站长干预"],
  "required_rules": ["必须说明报名按排名", "必须说明不是站长干预"],
  "forbidden_rules": ["禁止承诺一定获得资格"],
  "trigger_conditions": ["用户提到排名、名额、报不上"],
  "expected_final_state": "用户理解规则或结束咨询",
  "user_behavior_type": "情绪不满",
  "difficulty": "中等",
  "max_turns": 4,
  "data_source": "manual"
}
```

`GET /api/cases`：获取全部用例，后端按 `task_id + name + initial_message` 去重展示。

`GET /api/cases?task_id=1`：获取某个任务下的用例。

`GET /api/cases/{case_id}`：获取用例详情。

`PUT /api/cases/{case_id}`：更新用例。

`DELETE /api/cases/{case_id}`：删除用例。

`POST /api/cases/generate`：根据任务指令生成测试用例草稿，不直接入库。

生成草稿请求示例：

```json
{
  "task_id": 1,
  "case_count": 6,
  "difficulty_distribution": ["简单", "中等", "困难"],
  "user_behavior_types": ["正常配合", "拒绝配合", "情绪不满", "反复追问", "信息缺失", "超范围问题"]
}
```

`POST /api/cases/deduplicate`：手动清理历史重复用例，迁移 run/report/batch item 引用后保留最早记录。

## 自动评测

`POST /api/runs/start`

请求示例：

```json
{
  "task_id": 1,
  "case_id": 1,
  "model_provider": "mock_fallback"
}
```

返回示例：

```json
{
  "success": true,
  "run_id": 1,
  "report_id": 1,
  "total_score": 86.4,
  "provider_requested": "openai_compatible",
  "provider_used": "openai_compatible",
  "fallback_used": false,
  "model_provider": "openai_compatible",
  "model_name": "qwen-plus",
  "task_type": "rider_outbound",
  "message": "evaluation finished",
  "error_code": null,
  "error_message": null
}
```

若真实 provider 调用失败且 `TARGET_MODEL_ALLOW_FALLBACK=false`，接口返回 `success=false`、`run_id`、`error_code` 和 `error_message`，不静默回退到 mock。

`GET /api/runs/{run_id}`：获取执行记录。

`GET /api/runs/{run_id}/messages`：获取完整多轮对话记录。

`POST /api/runs/stream`：启动流式单次评测。接口以 SSE 方式返回 `assistant_start`、`assistant_delta`、`assistant_done`、`user_message`、`rule_result`、`report_generating`、`judge_result` 和 `done` 等事件，前端用于逐步展示多轮对话和报告生成状态。

`POST /api/runs/quick-check`：执行单条问答检测，不创建正式 run/report。

请求示例：

```json
{
  "task_id": 1,
  "case_id": 1,
  "model_provider": "openai_compatible",
  "user_messages": ["飞毛腿有什么好处", "我在开车"]
}
```

返回示例：

```json
{
  "success": true,
  "total_score": 100,
  "turns": [
    {
      "turn_index": 1,
      "user_message": "飞毛腿有什么好处",
      "assistant_message": "完成有助保资格，连续达标可获额外奖励。",
      "rule_score": 100,
      "matched_rules": ["说明完成有助保资格", "说明连续达标可能有额外奖励"],
      "failed_rules": [],
      "fact_checks": []
    }
  ]
}
```

## 批量评测

`POST /api/batch-runs/start`：按任务、用例、模型 provider 和重复次数启动批量评测。

`GET /api/batch-runs/{batch_id}`：查询批量任务详情。

`GET /api/batch-runs/{batch_id}/summary`：查询平均分、平均延迟、通过率、失败规则 TOP5 和报告列表。

## 模型配置

`GET /api/model-providers`：返回 `mock_fallback`、`openai_compatible`、`custom_endpoint` 三类 provider，不返回 API Key 明文。

`POST /api/model-providers/test`：测试 provider 连接状态。`openai_compatible` 会发起一次真实 OpenAI-compatible 调用并返回 `success=true`/`ok=true`；历史 `mock_baseline` / `mock_strong` 会兼容映射到 `mock_fallback`。

## 数据大屏

`GET /api/dashboard/summary`

返回任务、用例、run/report 统计、平均分、平均延迟、失败规则 TOP5、趋势数据和 AI 能力模块所需信息。
