# 接口说明

基础地址：

```text
http://127.0.0.1:8080
```

## 系统健康检查

`GET /health`

返回示例：

```json
{
  "status": "ok",
  "service": "callflow-lab"
}
```

## 评测任务

`POST /api/tasks`

请求示例：

```json
{
  "name": "外卖退款客服流程评测",
  "description": "用户投诉外卖超时并要求退款",
  "target_scenario": "外卖退款",
  "system_instruction": "客服必须核实订单号，不能直接承诺退款成功，需要告知处理时效，并安抚用户情绪。",
  "evaluation_goal": "评测退款投诉场景中的流程完整性和合规承诺。"
}
```

`GET /api/tasks`：获取任务列表。

`GET /api/tasks/{task_id}`：获取任务详情。

`PUT /api/tasks/{task_id}`：更新任务。

`DELETE /api/tasks/{task_id}`：删除任务。

## 测试用例

`POST /api/cases`

请求示例：

```json
{
  "task_id": 1,
  "name": "普通用户退款诉求",
  "user_profile": "普通用户，表达清楚，希望尽快退款。",
  "initial_message": "我的外卖迟到了快一个小时，饭都凉了，我想申请退款。",
  "max_turns": 4,
  "expected_goals": ["完成退款受理前置核实", "说明处理时效", "安抚用户情绪"],
  "required_rules": ["必须核实订单号", "必须告知处理时效", "必须安抚用户情绪"],
  "forbidden_rules": ["禁止直接承诺退款成功", "禁止跳过核实"],
  "difficulty": "简单"
}
```

`GET /api/cases`：获取全部用例。

`GET /api/cases?task_id=1`：获取某个任务下的用例。

`GET /api/cases/{case_id}`：获取用例详情。

`PUT /api/cases/{case_id}`：更新用例。

`DELETE /api/cases/{case_id}`：删除用例。

## 自动评测

`POST /api/runs/start`

请求示例：

```json
{
  "task_id": 1,
  "case_id": 1
}
```

返回示例：

```json
{
  "run_id": 1,
  "report_id": 1,
  "total_score": 96.0,
  "message": "evaluation finished"
}
```

`GET /api/runs/{run_id}`：获取执行记录。

`GET /api/runs/{run_id}/messages`：获取完整多轮对话记录。
