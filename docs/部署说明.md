# 部署说明

## 环境要求

- Python 3.10
- Anaconda 环境：`callflow-lab`
- Node.js 18 或更高版本
- npm
- Windows PowerShell 或其他终端

## 后端启动

进入项目后执行：

```powershell
conda activate callflow-lab
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8080
```

启动成功后访问：

```text
http://127.0.0.1:8080/docs
```

## 前端启动

打开新的终端，执行：

```powershell
cd frontend
npm install
npm run dev
```

启动成功后访问：

```text
http://127.0.0.1:5173
```

## 演示数据初始化

后端启动时会检查数据库是否已有任务，并执行兼容迁移。默认演示数据来自脱敏 Excel 指令；如果 Excel 文件不存在，会自动使用当前外呼任务 fallback 数据：

- 飞毛腿骑手合同生效外呼评测
- 课程直播产品升级外呼评测

fallback 数据必须保持外呼评测语义，不再使用早期旧客服样例。

任务判重优先使用 `name + task_type`；测试用例判重使用 `task_id + name + initial_message`。重复启动后端或重复导入指令时，不应重复创建同一用例。

## 模型配置

被测模型 provider 默认配置：

```env
TARGET_MODEL_PROVIDER=mock_fallback
TARGET_MODEL_API_KEY=
TARGET_MODEL_BASE_URL=
TARGET_MODEL_NAME=
TARGET_MODEL_ENDPOINT=
TARGET_MODEL_ALLOW_FALLBACK=false
```

`openai_compatible` 被测模型调用与 `scripts/test_qwen_api.py` 保持一致：使用 OpenAI SDK、60 秒超时，并从 `TARGET_MODEL_API_KEY`、`TARGET_MODEL_BASE_URL`、`TARGET_MODEL_NAME` 读取配置。`TARGET_MODEL_ALLOW_FALLBACK=false` 时真实 API 失败会直接返回错误，不静默回退。

用户模拟器配置：

```env
LLM_PROVIDER=mock
LLM_API_KEY=
LLM_BASE_URL=
LLM_MODEL=
```

评估器配置：

```env
EVALUATOR_PROVIDER=mock
EVALUATOR_API_KEY=
EVALUATOR_BASE_URL=
EVALUATOR_MODEL=
```

测试用例生成器配置：

```env
CASE_GENERATOR_PROVIDER=mock
CASE_GENERATOR_API_KEY=
CASE_GENERATOR_BASE_URL=
CASE_GENERATOR_MODEL=
```

`mock_fallback` 仅用于无 API Key 或外部服务不可用时兜底跑通流程，不代表真实 AI 能力。真实模型 Base URL、Model Name、Endpoint 和 API Key 仍以 `.env` 为配置来源；前端只展示非敏感配置和 API Key 是否已配置。

## 常见问题

### Swagger 无法访问

确认后端启动端口为 8080：

```powershell
uvicorn main:app --reload --port 8080
```

### 前端没有数据

确认后端已启动，并检查 `frontend/src/api/request.js` 中的默认地址是否为：

```text
http://127.0.0.1:8080
```

### npm install 失败

确认 Node.js 和 npm 已安装，并检查网络或 npm 源配置。可在网络稳定后重新执行 `npm install`。

### 数据为空

确认 `backend/callflow_lab.db` 是否已经存在且没有历史空数据。若需要重新生成演示数据，可在停止后端后备份并删除该文件，再重新启动服务触发自动初始化。

### 用例列表出现历史重复项

优先确认列表接口是否已经按 `task_id + name + initial_message` 去重展示。若需要清理历史重复记录，可调用：

```text
POST /api/cases/deduplicate
```
