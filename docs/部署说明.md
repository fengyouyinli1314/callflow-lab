# 部署说明

## 环境要求

- Python 3.10
- Anaconda 环境：`callflow-lab`
- Node.js 18 或更高版本
- npm
- Windows PowerShell 或其他终端

## 后端启动

进入项目后执行：

```powershell
conda activate callflow-lab
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8080
```

启动成功后访问：

```text
http://127.0.0.1:8080/docs
```

## 前端启动

打开新的终端，执行：

```powershell
cd frontend
npm install
npm run dev
```

启动成功后访问：

```text
http://127.0.0.1:5173
```

## 演示数据初始化

后端启动时会检查数据库是否已有任务。如果没有任务，会自动创建 3 个示例任务和 9 个测试用例：

- 外卖退款客服流程评测
- 酒店预订变更流程评测
- 到店团购券核销流程评测

如果需要重新生成演示数据，可停止后端后删除 `backend/callflow_lab.db`，再重新启动服务。

## 模型配置

默认配置：

```env
LLM_PROVIDER=mock
LLM_API_KEY=
LLM_BASE_URL=
LLM_MODEL=
```

接入 OpenAI compatible API：

```env
LLM_PROVIDER=openai_compatible
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://your-model-endpoint/v1
LLM_MODEL=your-model-name
```

未配置 API Key 或接口失败时，系统自动回退 mock 模式。

## 常见问题

### Swagger 无法访问

确认后端启动端口为 8080：

```powershell
uvicorn main:app --reload --port 8080
```

### 前端没有数据

确认后端已启动，并检查 `frontend/src/api/request.js` 中的默认地址是否为：

```text
http://127.0.0.1:8080
```

### npm install 失败

确认 Node.js 和 npm 已安装，并检查网络或 npm 源配置。可在网络稳定后重新执行 `npm install`。

### 数据为空

确认 `backend/callflow_lab.db` 是否已经存在且没有历史空数据。可删除该文件后重启后端触发自动初始化。
